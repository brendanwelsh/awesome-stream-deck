# awesome-stream-deck - context

## What & why
A curated "awesome" list of the best plugins, tools, libraries, icons, and resources for the Elgato Stream Deck. The goal is a hand-picked, high-quality index (not an exhaustive directory) that helps streamers, makers, automators, and plugin developers get the most out of the device.

## Where it fits
Standalone, content-only repository carrying the [Awesome](https://awesome.re) badge. It is the canonical curated list for the Stream Deck ecosystem; it links out to third-party projects and official Elgato resources rather than hosting any code itself.

## Run / build / test
No build, runtime, or test suite — the deliverable is `readme.md`. To validate before contributing, run the standard Awesome linter: `npx awesome-lint`. Check that all links resolve and that entries follow the required format.

## Layout & key files
- `readme.md` - the list itself; categorized sections with a matching `## Contents` table of contents at top.
- `contributing.md` - entry format and quality bar (must read before editing the list).
- `code-of-conduct.md` - community standards.
- `license` - project license.
- `media/logo.svg` - header logo referenced by `readme.md`.
- `.gitignore` - ignores node_modules, logs, OS files, editor folders.

## Gotchas
- Entry format is strict: `[Name](link) - Description.` — description starts with an uppercase letter and ends with a period.
- Do NOT write "Stream Deck" in entry descriptions; the whole list is already about it.
- Keep descriptions short and free of marketing language; check spelling/grammar.
- Add new entries at the bottom of the most appropriate category, and keep the `## Contents` anchors in sync when adding/renaming sections.
- Only include maintained, working, genuinely-best-in-class resources (archived/abandoned projects are removed; icon packs and hardware designs are evergreen exceptions).

## Status
Active and maintained. Working tree clean; list recently expanded with home automation, monitoring, and additional plugin categories.
